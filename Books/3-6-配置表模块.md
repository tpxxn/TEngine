# 3-6. 配置表模块 - ConfigSystem

## 📊 模块概述

TEngine 配置表模块集成了最佳游戏配置解决方案 **[Luban](https://github.com/focus-creative-games/luban)**，提供了完整的配置表开发工作流。

### ✨ 核心优势

1. **🚀 多种加载方式**
   - **懒加载**：按需加载配置，节省内存
   - **异步加载**：基于 UniTask 的异步加载，不阻塞主线程
   - **同步加载**：快速同步加载配置
   - **服务器加载**：支持 Task 异步加载（服务器端）

2. **📦 强大的数据支持**
   - 支持 Excel、JSON、XML、YAML 等多种数据格式
   - 支持复杂嵌套结构
   - 支持 OOP 类型继承

3. **✅ 完善的数据校验**
   - ref 引用检查
   - path 资源路径检查
   - range 范围检查
   - 类型安全检查

4. **🌍 本地化支持**
   - 静态文本值本地化
   - 动态文本值本地化
   - 时间本地化
   - main-patch 多地区版本

## 📁 目录结构

在 TEngine 中，Luban 配置表目录位于：

```
Configs/GameConfig/
├── CustomTemplate/      # 自定义模板
├── Datas/              # 配置表数据（Excel）
├── Defines/            # 配置表定义
└── gen_code_*.bat      # 代码生成脚本
```

生成产物路径：

- **二进制数据文件**：`Assets/AssetRaw/Configs/bytes/`（由资源模块加载）
- **C# 代码**：`GameScripts/HotFix/GameProto/GameConfig/`（热更程序集，Luban 生成）

## 🛠️ 安装与配置

### 1. 安装 Luban

在 TEngine 根目录同级克隆最新的 luban-next 仓库：

![image](src/3-6-2.png)

### 2. 构建 Luban

在 `Tools` 目录执行 `build-luban.bat`（Windows）或 `build-luban.sh`（Mac/Linux）：

![image](src/3-6-3.png)

### 3. 生成配置表代码

在 `Configs/GameConfig` 目录下执行对应的生成脚本：

- `gen_code_bin_to_project.bat` - 生成客户端代码（同步加载）
- `gen_code_bin_to_project_lazyload.bat` - 生成客户端代码（懒加载）
- `gen_code_bin_to_server.bat` - 生成服务器代码

> **提示**：TEngine 内置默认使用懒加载配置，也支持基于 UniTask 的异步加载、同步加载，包括服务器的 Task 异步加载。

## 📖 使用指南

### 基础使用

```csharp
/// <summary>
/// 道具配置表管理器。
/// </summary>
public class ItemConfigMgr : Singleton<ItemConfigMgr>
{
    /// <summary>
    /// 道具 Table。
    /// </summary>
    private TbItem TbItem => ConfigSystem.Instance.Tables.TbItem;

    /// <summary>
    /// 获取道具配置表。
    /// </summary>
    /// <param name="itemId">道具Id。</param>
    /// <returns>道具配置表。</returns>
    public ItemConfig GetItemConfig(int itemId)
    {
        if (TbItem.DataMap.TryGetValue(itemId, out var config))
        {
            return config;
        }
        return null;
    }
    
    /// <summary>
    /// 获取所有道具配置。
    /// </summary>
    /// <returns>所有道具配置列表。</returns>
    public List<ItemConfig> GetAllItemConfigs()
    {
        return TbItem.DataList;
    }
}
```

### 懒加载模式（推荐）

```csharp
// 懒加载：首次访问 Tables 属性时自动加载
var itemConfig = ConfigSystem.Instance.Tables.TbItem.Get(itemId);
```

### 同步加载模式

```csharp
// 同步加载所有配置表
ConfigSystem.Instance.Load();
var itemConfig = ConfigSystem.Instance.Tables.TbItem.Get(itemId);
```

## 🔧 Luban 核心特性

### 1. 强大的数据解析和转换能力

支持多种数据格式输入和输出：

**输入格式**：
- Excel (CSV, XLS, XLSX)
- JSON
- BSON
- XML
- YAML
- Lua
- Unity ScriptableObject

**输出格式**：
- Binary
- JSON
- BSON
- XML
- Lua
- YAML
- Erlang
- Custom Format

### 2. 增强的 Excel 格式

可以简洁地配置出：
- 简单列表
- 子结构
- 结构列表
- 任意复杂的深层次嵌套结构

### 3. 完备的类型系统

- 支持 OOP 类型继承
- 灵活优雅表达行为树、技能、剧情、副本等复杂 GamePlay 数据
- 支持生成 C#、Java、Go、C++、Lua、Python、JavaScript、TypeScript、Erlang、Rust、GDScript 代码

### 4. 强大的数据校验能力

- **ref 引用检查**：检查配置表之间的引用关系
- **path 资源路径检查**：检查资源路径是否存在
- **range 范围检查**：检查数值范围
- **类型安全检查**：编译时类型检查

### 5. 完善的本地化支持

- 静态文本值本地化
- 动态文本值本地化
- 时间本地化
- main-patch 多地区版本

### 6. 强大灵活的自定义能力

- 支持自定义代码模板
- 支持自定义数据模板
- 通用型生成和缓存工具
- 也可以用于生成协议、数据库之类的代码

## 💡 最佳实践

### 1. 使用懒加载模式

对于大型项目，推荐使用懒加载模式，按需加载配置：

```csharp
// ✅ 推荐：懒加载，按需加载
var config = ConfigSystem.Instance.Tables.TbItem.Get(itemId);
```

### 2. 创建配置表管理器

为每个配置表创建管理器，统一管理配置访问：

```csharp
public class ItemConfigMgr : Singleton<ItemConfigMgr>
{
    private TbItem TbItem => ConfigSystem.Instance.Tables.TbItem;
    
    public ItemConfig GetItemConfig(int itemId)
    {
        return TbItem.Get(itemId);
    }
}
```

### 3. 使用配置表验证

在配置表定义中使用验证特性：

```csharp
// 在 Luban 配置表中使用验证
item_id: int, ref: TbItem;  // 引用检查
item_count: int, range: [1, 999];  // 范围检查
item_path: string, path: "Assets/";  // 路径检查
```
